# v0.5 — Portal UI + evaluator wire-up

This release adds:

* **Backend**: `/assessments/{id}/report/summary` now runs the v0.3
  evaluator against collected evidence and returns real pass/fail
  totals, per-framework breakdowns, and findings. An HTML report is
  generated as part of every successful run.
* **Portal** (`portal/`): Next.js 14 web app with MSAL.js sign-in, a
  framework selector ("Start Assessment"), live status polling, and a
  results view with KPI cards + findings table.
* **CI**: new `Portal CI` workflow (typecheck, lint, test, build).

## What to do on your end

### 1. Extract the zip on a fresh branch

```powershell
cd C:\dev\compliance-collector
git checkout main
git pull origin main
git checkout -b v0.5-portal

Copy-Item "$env:USERPROFILE\Downloads\compliance-collector-v0.5.zip" .
Expand-Archive -Path .\compliance-collector-v0.5.zip -DestinationPath .\_v05_tmp -Force

# New folder
Move-Item .\_v05_tmp\portal .\portal -Force

# Overwrite the two backend files whose behavior changed (evaluator wire-up)
Move-Item .\_v05_tmp\backend\app\jobs.py .\backend\app\jobs.py -Force
Move-Item .\_v05_tmp\backend\app\routers\reports.py .\backend\app\routers\reports.py -Force

# New CI workflow
Move-Item .\_v05_tmp\.github\workflows\portal-ci.yml .\.github\workflows\portal-ci.yml -Force

# New doc
Move-Item .\_v05_tmp\docs\v0.5-merge.md .\docs\v0.5-merge.md -Force

Remove-Item .\_v05_tmp -Recurse -Force
Remove-Item .\compliance-collector-v0.5.zip
```

### 2. Add `portal` to the root ruff exclude

Open `pyproject.toml` at the repo root and update `[tool.ruff]`:

```toml
[tool.ruff]
line-length = 100
target-version = "py311"
exclude = ["backend", "infra", "portal"]
```

### 3. Smoke test the backend changes

```powershell
.\.venv\Scripts\Activate.ps1
cd backend
pytest
cd ..
```

All existing backend tests should still pass. The evaluator wire-up is
covered by the existing 409-when-not-completed test plus the core
library's evaluator tests.

### 4. Smoke test the portal

Install Node.js 20+ if you don't have it (<https://nodejs.org/>).

```powershell
cd portal
Copy-Item .env.example .env.local
npm install                 # first run pulls ~300MB of deps
npm run typecheck
npm run lint
npm test
npm run dev
```

Open <http://localhost:3000>. In a second terminal, make sure the
backend is running:

```powershell
cd C:\dev\compliance-collector\backend
.\..\.venv\Scripts\Activate.ps1
uvicorn app.main:app --reload --port 8080
```

Dev bypass is on by default — you can click "Start assessment", pick
CIS M365, and submit. The backend will queue the run. (Without real
Graph credentials the collectors will fail, but the end-to-end
plumbing — create, poll, show status — works.)

### 6. Open PR and wait for CI

Three workflows should run on the PR:

* **CI / test (3.11, 3.12)** — core library (unchanged, should pass).
* **Backend CI / test (3.11, 3.12)** — installs core + backend, runs
  ruff + pytest. The evaluator wire-up is tested by the existing
  backend tests.
* **Portal CI / build** — installs npm deps, typecheck, lint, vitest,
  and a production Next.js build.

If Portal CI fails, the most likely causes are:

* **`npm ci` fails on missing lockfile**: the workflow already falls
  back to `npm install`, so first PR will use that. After merge, check
  in the `package-lock.json` npm generated locally and future runs will
  use `npm ci`.
* **ESLint flags something in a generated page**: run `npm run lint`
  locally, copy the rule names, I'll patch.
* **Type error**: paste the log.

### 7. Merge and tag

Once all checks are green:

1. **Merge pull request** → **Confirm merge** → **Delete branch**.
2. Locally:
   ```powershell
   git checkout main
   git pull origin main
   git branch -d v0.5-portal
   git tag -a v0.5.0 -m "v0.5.0: portal UI + evaluator wire-up"
   git push origin v0.5.0
   ```
3. On GitHub, draft a release from tag `v0.5.0`.
