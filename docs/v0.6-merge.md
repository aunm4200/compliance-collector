# v0.6 — merge & release (Azure DevOps)

## What's in this version

* **`portal/Dockerfile` + `.dockerignore`** — production container image,
  Next.js standalone output, non-root user, healthcheck.
* **`portal/next.config.mjs`** — adds `output: "standalone"` for Docker.
* **`infra/main.bicep`** — adds portal Container App, `AcrPull` role for
  the UAMI, CORS on backend ingress, richer outputs.
* **`azure-pipelines/deploy.yml`** — manual build + push + roll out
  pipeline, uses a workload-identity service connection (no secrets).
* **`azure-pipelines/ci.yml`** — optional replacement for the GitHub
  Actions CI workflows.
* **`azure-pipelines/README.md`** — quick pipeline reference.
* **`docs/v0.6-deploy.md`** — full Azure DevOps setup walkthrough.
* **`docs/demo-script.md`** — 10-minute demo runbook.

## Removed

* `.github/workflows/deploy.yml` — superseded by `azure-pipelines/deploy.yml`.

## Merge checklist

```powershell
git checkout -b v0.6-azure-hosting

git add portal/Dockerfile portal/.dockerignore portal/next.config.mjs `
        infra/main.bicep `
        azure-pipelines/deploy.yml azure-pipelines/ci.yml azure-pipelines/README.md `
        docs/v0.6-deploy.md docs/demo-script.md docs/v0.6-merge.md

git rm .github/workflows/deploy.yml

git commit -m "v0.6: Azure Container Apps hosting with Azure DevOps CI/CD"
git push -u origin v0.6-azure-hosting

gh pr create --fill   # or open a PR in Azure Repos
```

After CI is green and the PR is merged:

```powershell
git checkout main
git pull
git tag -a v0.6.0 -m "v0.6.0 — Azure hosting, Azure DevOps CI/CD"
git push origin v0.6.0
```

## First deploy

Follow `docs/v0.6-deploy.md` top to bottom. Rough timing: 30 minutes
end-to-end, most of it waiting for the Bicep deployment and the first
pipeline run.
